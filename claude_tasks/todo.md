# 茶园扫码导览网站 — TODO

## 目标
游客在景点扫描二维码 → 打开网页 → 弹窗显示该景点的介绍和图片 → 关闭弹窗后看到整张景区导览图 `map.jpg`。

## 决策（已与 Eva 确认）
- 景点名称：从 `map.jpg` 提取真实名称；介绍/图片用占位内容。
- 技术栈：纯静态 HTML/CSS/JS，无构建步骤、无框架。
- 二维码：附带生成工具，可打印每个景点的二维码。

## 架构
- `index.html` — 主页面：全屏展示导览图；读取 URL `?spot=ID`，自动弹出对应景点弹窗。
- `css/style.css` — 样式。
- `js/spots.js` — 景点数据（id / name / intro / images）。
- `js/app.js` — 逻辑：解析 URL、渲染弹窗、关闭后显示地图。
- `assets/placeholders/` — 每个景点的占位图片（SVG）。
- `qr/index.html` — 二维码生成/打印工具（本地内置 qrcode 库，离线可用）。
- `README.md` — 使用与部署说明。

## 追加需求：移动端自适应（已完成）
- 受众主要为手机用户，整体改为移动端优先、按屏幕尺寸自适应。
- viewport 放开缩放（原 `user-scalable=no` 已移除）+ `viewport-fit=cover` + theme-color；地图细节多，手机可双指放大。
- 弹窗在 ≤600px 改为**底部抽屉**（圆角顶 + 小手柄 + 上滑动画），并实现**下滑关闭**手势（避开内容滚动冲突）。
- 适配刘海/手势条安全区（`env(safe-area-inset-*)`），高度用 `dvh` 防地址栏遮挡；顶部提示条窄屏自动换行。
- 画廊图片高度自适应 `clamp(180px,52vw,260px)`；横屏手机弹窗回到居中并压缩图高。
- 编辑器在 ≤720px 改为上下堆叠，手机也可用。
- 验证：Chrome headless 截图 390×844（地图/底部抽屉）、横屏 780×390、编辑器窄屏，布局均正常。
